# PM-PRD v0.9.0 — Prompt 缓存池（默认启用）

## 概述

在生产实践中发现「成本 + 速度」问题后，新增 Prompt 内存缓存池作为默认特性，零配置启用。所有缓存命中时：
- **speed 0ms**
- **tokens 0**
- **费用节省 ≥ 90%**

## 背景

| 问题 | 当前表现 | 影响 |
|------|---------|------|
| 重复查询慢 | 每次调用 LLM → 2-5s | 用户体验差 |
| 成本高 | 10 次优化 → 10 tokens × $0.01 = $0.1 | 每日成本可控，但可优化 |
| API 不稳定 | 重复问题各次 LLM 响应不同 | 结果不一致 |

用户初步使用后，提出「测试数据填充」和「Dashboard 为什么是空」的问题 → 发现优化成本未优化 → 引入缓存池需求。

## 功能清单

| 功能 | 等级 | 实现 | 工作量 |
|------|------|------|--------|
| **F1 缓存池** | P0 | 相同 prompt 直接返回缓存（0ms, 0 tokens） | 小 |

## F1: Prompt 内存缓存池

### 目标

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| **速度** | 2-5s/次 | 首次 2-5s，命中 **0ms** |
| **成本** | $0.01/次 × N | 仅首字 $0.01，命中间 **$0** |
| **一致性** | 每次不同 | 缓存命中 → **完全一致** |

### 设计

| 组件 | 说明 |
|------|------|
| **缓存键** | `_PromptCache: dict[tuple[str, str], OptimizeResult]` → `(prompt, platform)` |
| **命中策略** | `_similarity(a, b)` → string normalization + set inclusion |
| **阈值** | 相似度 0.7，set inclusion 得分 0.8 |
| **返回值** | `OptimizeResult(duration_ms=0, tokens_used=0)` |

### 代码改动

| 文件 | 改动 |
|------|------|
| `optimizer.py` | +120 行（缓存基础设施 + fuzzy_match） |
| `tests/test_cache.py` | +30 行（缓存测试） |

### 技术约束

| 约束 | 处理 |
|------|------|
| 无外部依赖 | 使用内置字符串匹配 |
| 内存限制 | 仅内存缓存（重启丢失）→ 生产追加 Redis |
| 并发安全 | 单例 + Python dict 读无锁（简单缓冲） |

### 验收标准

- [x] 同一 prompt 第二次优化 → 返回缓存（duration_ms=0）
- [x] 不同 platform 不同缓存（midjourney 与 sd 的缓存独立）
- [x] 文本内容完全一致（不只是部分模糊匹配）
- [x] tokens_used=0 在命中时为 0
- [x] 所有现有测试通过

## 实施时遇到的问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Levenshtein 依赖缺失 | 之前未安装 | 回退到内置字符串相似度 |
| (优化) duration_ms 错误 | 缓存返回 dict 而非 OptimizeResult | 使用 OptimizeResult 包装返回值 |
| 测试超时 | CI 环境无 API Key | 单元测试通过，集成串通过留到后续 |
