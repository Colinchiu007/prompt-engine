# PM-PRD v0.16.0 — 输入验证 + 短文本处理

## 概述

用户输入「好吧」等短文本时，LLM 生成无关图像 —— 因为 prompt 没有明确主题。

## 方案

### F1: 输入验证

API 层加验证：

| 规则 | 说明 |
|------|------|
| 最短长度 | 中文 ≥ 3 字，英文 ≥ 3 词 |
| 返回 | 400 + 提示「描述太简短，建议更详细描述画面」 |

### F1.5: system prompt 改进

在 MJ/SD/DALL·E 等策略的 system prompt 中新增：

```
## 短文本处理
如果用户输入的描述非常简短（< 5 个字或词），不要自动生成无关画面。
此时：保留输入原意，围绕输入创作明确的视觉。
对"好吧"等无视觉含义的输入，返回基于输入主题的最小增量扩展。
```

### F2: 前端提示

当验证失败时，前端显示友好的中文提示：「描述太简短了！试试更详细的描述，比如"一只猫坐在天鹅绒宝座上"」

## 验收

- [x] 输入"好吧" → 400 错误 + 友好提示
- [x] 输入"a cat" → 仍能优化（≥ 3 词）
- [x] system prompt 新增短文本处理规则
